# pi-workflow

A planned Pi extension for deterministic, role-aware workflow lifecycles with Markdown planning artifacts and direct interagent handoffs.

The planning baseline is complete; implementation has not started. See:

- [`PLAN.md`](./PLAN.md) for the approved V1 direction and implementation sequence.
- [`IDEAS.md`](./IDEAS.md) for explicitly deferred possibilities.

V1 will focus on the Architect, Sergeant, and Worker workflows already proven in Practorium. It will use a coarse-grained finite-state machine for lifecycle ownership and gates while leaving substantial planning and execution guidance in Markdown.
